# IDC MTR 全路由節點監測工具（Windows 版）

利用 Windows 內建的 `tracert` + `ping` 指令，自動掃描來源到目標 IP 的所有路由跳點，
持續記錄網路品質（延遲、封包遺失率），並產生視覺化 HTML 報告。
**不需要安裝任何第三方網路工具。**

---

## 檔案說明

| 檔案 | 說明 |
|------|------|
| `IDC_MTR_Monitor.exe` | 可直接執行的 Windows 執行檔（免安裝 Python） |
| `idc_mtr_monitor_win.py` | Python 原始碼 |

執行後會自動產生以下檔案（以目標 IP `8.8.8.8` 為例）：

| 檔案 | 說明 |
|------|------|
| `idc_mtr_win_report_8-8-8-8.html` | HTML 監測報告（瀏覽器開啟） |
| `idc_mtr_win_8-8-8-8.db` | SQLite 資料庫（歷史紀錄） |
| `idc_mtr_win.log` | 執行日誌 |

---

## 快速開始

### GUI 模式（推薦）

直接雙擊 `IDC_MTR_Monitor.exe`，彈出設定視窗，填入目標 IP 及參數後按「開始監測」。

### 命令列模式（進階）

```
IDC_MTR_Monitor.exe 8.8.8.8
IDC_MTR_Monitor.exe 168.95.1.1 -i 600 -c 20
```

完整參數：

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `target` | `8.8.8.8` | 目標 IP |
| `-i, --interval` | `300` | 掃描間隔（秒） |
| `-c, --cycles` | `10` | 每跳 Ping 次數 |
| `-w, --workers` | `8` | 平行執行緒數 |
| `-r, --report-every` | `1` | 每 N 次掃描更新報告 |
| `--hours` | `24` | 報告最小涵蓋時數 |
| `--db` | 自動 | 自訂資料庫路徑 |
| `--report` | 自動 | 自訂報告路徑 |

---

## 設定參數說明

| 參數 | 預設值 | 說明 |
|------|--------|------|
| 目標 IP | `8.8.8.8` | 要監測的目標 IP 位址 |
| 掃描間隔(秒) | `300` | 每次掃描間隔（建議 300 秒） |
| Ping 次數 | `10` | 每跳點發送封包數（建議 10~30） |
| 遺失警報% | `5.0` | 封包遺失率超過此值觸發警報 |
| 延遲警報ms | `150.0` | 平均延遲超過此值觸發警報 |

## 網路狀態判定邏輯

| 狀態 | 顏色 | 觸發條件 |
|------|------|----------|
| 正常 | 綠色 | 目標節點封包遺失 < 5% 且延遲 < 150ms，中繼無異常 |
| 注意 | 黃色 | 目標正常，但中繼節點遺失 >= 5% 或延遲 >= 150ms |
| 異常 | 紅色 | 目標節點封包遺失 >= 5% 或延遲 >= 150ms |

- 中繼節點 100% loss 但目標可達 → 判定為 ICMP 禁用，自動跳過不計
- ICMP 禁用節點在報告中以灰色標註，不影響整體狀態判定

## 注意事項

1. Log 和報告會存在 exe 所在的資料夾
2. 監控主機需要有網路連線才能執行 tracert/ping
3. 若 Windows Defender 誤判 exe，請加入排除清單
4. 停止程式：工作管理員 → 找到 `IDC_MTR_Monitor.exe` → 結束工作
5. 報告可直接用瀏覽器開啟 HTML 檔案查看，包含即時圖表
6. 資料庫 (.db) 保留所有歷史紀錄，可用 SQLite 工具查詢
