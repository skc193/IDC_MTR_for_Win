===================================================================================================
  IDC MTR 全路由節點 24 小時監測工具（Windows 版）使用說明
  版本：v2.0 | 更新日期：2026-03-15
===================================================================================================

【工具簡介】
  利用 Windows 內建的 tracert + ping 指令，自動掃描來源到目標 IP 的所有路由跳點，
  持續記錄網路品質（延遲、封包遺失率），並產生視覺化 HTML 報告。
  不需要安裝任何第三方網路工具。

===================================================================================================
【檔案說明】
===================================================================================================

  idc_mtr_monitor_win.exe     — 可直接執行的 Windows 執行檔（免安裝 Python）
  idc_mtr_monitor_win.py      — Python 原始碼

  執行後會自動產生以下檔案（以目標 IP 8.8.8.8 為例）：
  idc_mtr_win_report_8-8-8-8.html  — HTML 監測報告（瀏覽器開啟）
  idc_mtr_win_8-8-8-8.db           — SQLite 資料庫（歷史紀錄）
  idc_mtr_win.log                  — 執行日誌

===================================================================================================
【快速開始】
===================================================================================================

  方式一：GUI 模式（推薦新手使用）
  ─────────────────────────────────
  直接雙擊 idc_mtr_monitor_win.exe
  → 彈出設定視窗，填入目標 IP 及參數後按「開始監測」
  → 視窗可自由拖曳調整大小

  方式二：命令列模式（進階使用）
  ─────────────────────────────────
  idc_mtr_monitor_win.exe 8.8.8.8
  idc_mtr_monitor_win.exe 168.95.1.1 -i 600 -c 20

  完整參數：
    target                目標 IP（預設 8.8.8.8）
    -i, --interval N      掃描間隔秒數（預設 300）
    -c, --cycles N        每跳 Ping 次數（預設 10）
    -w, --workers N       平行執行緒數（預設 8）
    -r, --report-every N  每 N 次掃描更新報告（預設 1）
    --hours N             報告涵蓋時數（預設 24）
    --db PATH             自訂資料庫路徑
    --report PATH         自訂報告路徑

===================================================================================================
【設定參數說明】
===================================================================================================

  參數            預設值    說明
  ──────────────  ────────  ──────────────────────────────────
  目標 IP         8.8.8.8   要監測的目標 IP 位址
  掃描間隔(秒)    300       每次掃描的間隔時間（建議 300 秒）
  Ping 次數       10        每個跳點發送的 Ping 封包數（建議 10~30）
  遺失警報%       5.0       封包遺失率超過此值觸發警報（0~100）
  延遲警報ms      150.0     平均延遲超過此值觸發警報

===================================================================================================
【網路狀態判定邏輯】
===================================================================================================

  報告中的「整體狀態」依據以下三級判定：

  狀態      顏色    觸發條件
  ────────  ──────  ──────────────────────────────────────────────
  正常      綠色    目標節點封包遺失 < 5% 且延遲 < 150ms，中繼節點無異常
  注意      黃色    目標正常，但中繼節點遺失 >= 5% 或延遲 >= 150ms
  異常      紅色    目標節點封包遺失 >= 5% 或延遲 >= 150ms

  ※ 門檻值依設定畫面的「遺失警報%」和「延遲警報ms」而定

  判斷順序：
  1. 先看最終目標節點（最後一跳）→ 超標直接判定「異常」
  2. 目標正常時，檢查中繼節點 → 有超標判定「注意」
  3. 中繼節點 100% loss 但目標可達 → 判定為 ICMP 禁用，自動跳過不計

  ICMP 禁用節點說明：
  某些路由器會禁止 ICMP 回應，導致該跳點顯示 100% loss。
  程式會自動識別：若目標可達但中繼 100% loss，該節點在報告中以灰色淡化
  顯示並標註「ICMP 禁用」，不影響整體狀態判定。

===================================================================================================
【背景執行方式】
===================================================================================================

  方式一：背景執行 exe（無 GUI 視窗）
  ─────────────────────────────────
  start "" "idc_mtr_monitor_win.exe" 8.8.8.8

  停止：
  taskkill /IM idc_mtr_monitor_win.exe /F

  方式二：用 pythonw 背景執行（需安裝 Python）
  ─────────────────────────────────
  pythonw "idc_mtr_monitor_win.py" 8.8.8.8

  確認是否執行中：
  tasklist | findstr python

  停止：
  taskkill /IM pythonw.exe /F

  方式三：啟動腳本（推薦，記錄 PID 精準控制）
  ─────────────────────────────────
  建立 start_monitor.bat：

    @echo off
    start /B pythonw "idc_mtr_monitor_win.py" 8.8.8.8
    for /f "tokens=2" %%a in ('tasklist /FI "IMAGENAME eq pythonw.exe" /NH') do (
        echo %%a > "%~dp0monitor.pid"
        goto :done
    )
    :done
    echo 監測已啟動，PID 已存入 monitor.pid

  停止：
    for /f %%a in (monitor.pid) do taskkill /PID %%a /F

  方式四：Windows 工作排程器（開機自動啟動）
  ─────────────────────────────────
  1. 開啟「工作排程器」→ 建立基本工作
  2. 觸發程序：電腦啟動時
  3. 動作：執行程式 pythonw.exe
  4. 引數："idc_mtr_monitor_win.py" 8.8.8.8

===================================================================================================
【注意事項】
===================================================================================================

  1. Log 和報告會存在 exe 所在的資料夾
  2. 監控主機需要有網路連線才能執行 tracert/ping
  3. 若 Windows Defender 誤判 exe，請加入排除清單
  4. 停止程式：工作管理員 → 找到 idc_mtr_monitor_win.exe → 結束工作
  5. 報告可直接用瀏覽器開啟 HTML 檔案查看，包含即時圖表
  6. 資料庫 (.db) 保留所有歷史紀錄，可用 SQLite 工具查詢
